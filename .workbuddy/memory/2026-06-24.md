## 修复：对话选项分支后 AI 自由对话不显示

### 根因
`handleSelectOption` 在玩家点击选项后直接调用 `onAdvanceDialogue` 跳转到下一段，跳过了 AI 分支回应阶段。
对话框设计文档中每个 `playerOptions` 下方都有 `【AI生成指引】`，要求选选项后先生成 NPC 的上下文回应。

### 修复方案 (DialogueOverlay.jsx)
1. `handleSelectOption(optionText)` 改为接收选项文本，不再直接推进
2. 有 AI 时：切换到 AI 模式，以选项文本为用户消息，注入 NPC 对话上下文，流式生成 NPC 回应
3. 用 `branchAdvanceRef` 标记本次进入 AI 的来源（选项分支 vs 手动切换）
4. AI 流式完成后显示金色「继续剧情 ▶」按钮，点击后推进到下一段
5. 玩家也可在 AI 回应后继续自由对话，再点「继续剧情」推进
6. 无 AI 时保持原有行为（直接推进）

## 合并 transition 模式——选项分支对话新方案

### 根因（彻底解决）
之前的方案用 AI 自由对话模式（mode='ai'）承载分支回应，与固定对话的模式切换有竞态。新方案引入独立 `transition` 过渡模式，与 'fixed' / 'ai' 完全隔离。

### 核心改动 (DialogueOverlay.jsx)
1. **transition 状态**：`{ playerChoice, aiText, streaming, error, nextDlg }`，独立于 mode
2. **handleSelectOption 重写**：AI 融合玩家选项 + 下一段对话内容，生成连贯自然回应
3. **handleTransitionContinue**：过渡完成后推进到下一段，标记 skipNextLinesRef 跳过已表达的台词
4. **skipNextLinesRef**：useEffect 检测后跳过固定台词行
5. **移除 branchAdvanceRef / isBranchMode / canBranchAdvance** 全部分支模式代码

### CSS 新增
- `.dialogue-overlay__transition-*` 过渡模式完整样式（玩家气泡、NPC气泡、说话者标签）
- `.dialogue-overlay__options` / `.dialogue-overlay__option` 过渡模式「继续」按钮

### 保留的功能
NPC 立绘、音效、AI 自由对话、信任阶段记录、首次引导、所有 props 兼容。

### 根因
`handleExitAi` 中 `setMode('fixed')` 和 `onAdvanceDialogue(nextId)` 在同一微任务执行，React 批处理时 mode 变更和 dialogue 变更一起提交。下一个对话节点加载时 mode 状态不确定，导致第二个分支的 `handleSelectOption` 模式切换被干扰。

### 修复
将 `onAdvanceDialogue` 包装在 `Promise.resolve().then()` 中，确保 React 先提交 `mode='fixed'`，再推进对话。连续分支链 A→B→C 全部正常。

1. 撤销上述6个组件的 `return;` 短路
2. 每个组件的 `handleMouseDown` 入口加 `localStorage.getItem('shanglu_edit_mode') === '1'` 检查
3. Scene.jsx 新增编辑模式状态 + 右下角 🔒/🔧 切换按钮（持久化到 localStorage）
4. 保存布局按钮仅在编辑模式下显示
5. 左侧键长按旋转同样受编辑模式控制
6. 默认关闭编辑模式 → 玩家正常游玩不受影响
