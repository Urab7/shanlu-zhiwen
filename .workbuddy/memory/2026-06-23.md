### 完成内容
在 `src/ai/prompts/` 目录下创建了4个NPC prompt文件，加上之前已有的 `gesang.js`，共5个文件齐全：

1. **baima.js** — 白玛（修行者，约65岁，全村节奏最慢，经文与日常话混着说）
   - systemPrompt 取自 `baima_npc_test.py` 的 `BAIMA_SYSTEM_PROMPT`
   - 核心机制：≥3轮对话+触碰关键词 → 自动念经文（佛青制法+六字真言）
   - 知识边界：骨白只说"她留下的东西"，天葬完全不说，唐卡去向只说"不在了"

2. **danzeng.js** — 旦增·洛桑（唐卡画师，约70岁，十二年不与外人说话）
   - systemPrompt 取自 `danzeng_npc_test.py` 的 `DANZENG_SYSTEM_PROMPT`
   - 核心机制：禁忌词系统（唐卡/画/笔记/研究 → 立即关闭"你走吧。今天。"）
   - 先听后问测试：两轮不碰禁忌 → 第三轮"明天再来"
   - 每句≤15字，4-8秒停顿

3. **renqing.js** — 仁青（外来研究者，约35岁，全村节奏最快）
   - systemPrompt 取自 `renqing_npc_test.py` 的 `RENQING_SYSTEM_PROMPT`
   - 核心机制：自我约束"不合适说"，照片系统（5张分3章），门槛"你得让我看看你懂不懂唐卡"
   - 停顿1秒，短句多，偶尔打断自己

4. **zhuoma.js** — 卓玛（村长，约40岁，旦增的女儿）
   - systemPrompt 取自 `zhuoma_npc_test.py` 的 `ZHUOMA_SYSTEM_PROMPT`
   - 知道全部真相但绝不主动说——"那件事，不是我该说的"
   - 用行为结束对话（站起来拿文件）

### 文件结构
每个文件遵循 `gesang.js` 的格式：
- `export const {npcId} = { name, role, personality, knowledge[], speechStyle, forbidden[], systemPrompt }`
- `systemPrompt` 为完整模板字符串，直接喂给 LLM
- `forbidden` 数组取自 Python 脚本的 `FORBIDDEN_KEYWORDS`
- `knowledge` 数组综合自 MD 框架和 Python 脚本的 system prompt 内容
